# 관계 추출 정책 (연구 우선)

## 목적
- 본 프로젝트는 노드/엣지 개수 확대를 목표로 두지 않는다.
- 관계 데이터는 실제 영향 관계가 성립하는 경우에만 채택한다.

## 판정 상태
- `채택`: 데이터 파일에 반영 가능한 관계.
- `보류`: 추가 근거가 필요해 데이터 반영을 유예한 관계.
- `기각`: 현재 기준에서 관계가 성립하지 않아 반영하지 않는 관계.

## 채택 분류 체계 (Accepted Tracks)
`채택`은 아래 3개 트랙 중 하나로 분류한다.

1. `pioneer`
- 새로운 시스템(또는 당시 비주류였던 핵심 시스템)을 정립했고, 이후 여러 작품으로 확산된 게임.
- 요구 조건:
  - 시스템 정의를 명사구로 고정 가능해야 함.
  - 영향 방향(A -> B)을 직접 인용 가능한 근거로 고정 가능해야 함.
  - 후행 영향이 단일 후속작이 아닌 다수 작품(복수 스튜디오/프랜차이즈 포함)에서 확인돼야 함.

2. `popularizer`
- 최초 도입자는 아니지만, 시스템을 대중화/표준화한 게임.
- 요구 조건:
  - 선행 사례가 있다는 점이 확인돼야 함(창시 아님).
  - 해당 게임 이후 채택/확산 흐름이 확인돼야 함.
  - 대중화 근거(판매/이용자 규모/커뮤니티 파급/매체 분석)와 시스템 영향 근거가 함께 있어야 함.

3. `end_node`
- 최근작으로 후행 영향은 아직 확정되지 않았지만, 신규 시스템 또는 고품질 융합으로 향후 영향 가능성이 높은 게임.
- 시간 기준:
  - 2026-02-18 기준, 2023-01-01 이후 출시작을 기본 대상으로 본다.
- 요구 조건:
  - 신규성 또는 융합성이 명확히 설명돼야 함.
  - 유명성/인기도 신호가 확인돼야 함.
  - 후행 영향 미확정 상태임을 명시한다.
  - 그래프 반영 시 원칙적으로 끝 노드(outgoing edge 0)로 관리한다.

## 채택 기준
1. 메커닉/시스템 영향
- 전투, 성장, 탐험, 경제, 조작 리듬 등 핵심 플레이 구조에서 영향 관계가 설명 가능해야 한다.

2. 설계 철학 영향
- 난이도 철학, 인터페이스 철학, 레벨/퀘스트 설계 원칙 등에서 계승/변형이 확인되어야 한다.

3. 출처 강도
- 최소 1개 이상의 신뢰 가능한 1차/준1차 근거(공식 인터뷰, 포스트모템, 공식 문서, 고품질 분석)가 필요하다.
- 위키/저널 문서는 보조 근거로 사용하되, 문맥 일치 여부를 내부 검토로 확인한다.
- 주의: 채택 판정은 `출처 개수`가 아니라 `근거 품질`(방향성, 직접 인용 가능성, 시스템 정의 가능성)로 수행한다.

## 보류 기준
- 영향 후보는 합리적이지만, 근거가 간접적이거나 출처 강도가 낮은 경우.
- 동일 용어를 쓰더라도 실제 설계 의미가 다를 수 있어 검증이 필요한 경우.

## 기각 기준
- 단순 시리즈 계보 연결(후속작/리메이크/연도 순)만으로 제시된 경우.
- 마케팅 문구나 커뮤니티 추정만으로 영향 관계를 단정하는 경우.
- `확장|진화|정교화|후속작` 키워드 수준 설명에 머무르는 경우.

## 데이터 반영 원칙
- `채택` 항목만 `src/data/edges.json`에 반영한다.
- 노드는 채택 엣지와 연구 맥락에 따라 추가/유지/제거한다.
- 수량 상한/하한은 두지 않는다.
- 후보 검토 데이터(`docs/research/candidate-games.json`)에는 각 항목의 `acceptanceTrack`과 결손 근거를 남긴다.
- 전체 저장 노드(`src/data/nodes.json`)는 별도 전수 재검토 문서(`docs/research/stored-games-track-review.md`)로 주기 점검한다.
- 저장 노드 전수 재검토는 `npm run rebuild:stored-review`로 재생성한다.

## 문서 반영 원칙
- `docs/research/edge-evidence.md`: 채택된 관계 중심 근거 기록.
- `docs/research/sources.md`: 실제 사용한 출처만 기록.
- `docs/research/image-audit.md`: 이미지 선택/제외 사유 기록.

## 예시
- 채택 예시: "사망 리스크-회수 루프가 후속 작품의 핵심 진행 구조로 계승됨"처럼 플레이 구조 영향이 명확한 경우.
- 보류 예시: 장르가 유사하지만 개발 문맥/메커닉 연결이 약해 추가 근거가 필요한 경우.
- 기각 예시: "같은 시리즈의 최신작"이라는 이유만으로 연결한 경우.
